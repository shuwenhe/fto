# 检索+排序验收清单 v1

版本日期：2026-03-31
适用范围：FTO 专利检索服务（双路召回 + 融合排序）

## 1. 目标

- 检索与排序结果可复现、可解释、可回归。
- 离线评测与线上接口输出顺序一致（同数据、同参数、同 query）。

## 2. 验收入口

- 离线指标评测：
  - 命令：node scripts/eval_retrieval.mjs --k 5
- 线上/离线一致性：
  - 命令：node scripts/compare_online_offline.mjs --k 5 --sample 5 --seed 20260331 --base-url http://127.0.0.1/fto/api

## 3. 指标门槛（功能验收）

固定 k=5，数据快照固定后，需同时满足：

- Recall@5 >= 0.80
- MRR@5 >= 0.95
- NDCG@5 >= 0.90

说明：门槛按当前数据规模设置，后续数据扩容后可上调。

## 4. 稳定性定义（必须全部通过）

### 4.1 排序稳定性

- 对固定 query 集合（queries.jsonl），在固定 seed 条件下，线上 top-k 与离线 top-k 顺序完全一致。
- 一致性回归结果必须：passed = sample 且 failed = 0。

### 4.2 结果可复现性

在以下条件不变时，结果必须一致：

- 数据文件哈希不变（patents/queries/qrels）。
- 排序参数不变（k、融合权重、召回深度规则）。
- 后端代码版本不变。

### 4.3 运行稳定性

- 同一命令连续执行 3 次，不出现超时/失败。
- 如出现失败，必须输出 query_id 和在线/离线 top-k 差异明细。

## 5. 失败判定

任一条件不满足即验收失败：

- 指标低于门槛。
- 一致性检查 failed > 0。
- 连续执行出现任务失败、超时、空结果异常。
